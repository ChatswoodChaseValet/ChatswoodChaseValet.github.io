Features

Turn customer-facing services on or off. When a service is off it disappears from the staff dashboard and from the customer's SMS ticket page (no buttons, no alarms). Synced to all devices within seconds.

Get a notification on this phone when a customer requests their car, or when the desk assigns you a hands-free shopping pickup — even when the dashboard isn't open. Turn this on once per device, and stay logged in as yourself so assigned-job alerts reach you.

Auto-Save to File

Save your records to a file on this PC automatically — every time you log a car, edit, or check out a vehicle. Recommended: save the backup file inside your OneDrive, Google Drive, or Dropbox folder so it also gets cloud-backed.

○ Auto-save off — using browser storage only
How it works:
  • Chrome / Edge: Silent file save to a folder of your choice (recommended)
  • Other browsers: Auto-download to Downloads folder (look for the latest chatswood-valet-backup.json)
  • If you restart the browser, click Re-enable once to restore permission
  • Auto-save runs in addition to browser storage — you have two copies for safety

Usage this month: 0 scans · Resets monthly
Security: Token is visible in the dashboard's source code. Set spending cap to $0 in your Plate Recognizer account to prevent surprise charges.
AI Vehicle Identification
—

When staff use Snap, the photo is also read by AI to auto-fill Brand, Model and Colour — roughly a fraction of a cent per car, with no minimum spend. Results are cached per plate, and returning cars fill from your own visit history for free, so you only ever pay for genuinely new cars.

One-time setup: this needs the identify-vehicle function deployed with an Anthropic API key set as a Supabase secret. See the deploy notes at the top of supabase/functions/identify-vehicle/index.ts. Until that's done the toggle is harmless — scanning still works, it just won't auto-fill the car details.
Square Terminal (card payments)
—

Send the amount straight to a Square Terminal at checkout — the customer taps their card and the entry is marked Paid · Card automatically. Currently set up for Sandbox (test mode), so no real money moves until you switch to a live token.

One-time setup (Sandbox):
1. Create a free account at developer.squareup.com and open the Sandbox.
2. Set the access token as a secret: supabase secrets set SQUARE_ACCESS_TOKEN=EAAA… --project-ref ebqiitxiyzzbkgyfypss, then deploy square-terminal-checkout.
3. To test without hardware, use Square's sandbox auto-complete device ID: 9fa747a2-25ff-48ee-b078-04381f7c828f (paste it above). Note: this test device only approves amounts up to $25 — charge a short-stay car for testing; bigger amounts stay "Waiting" on purpose. A real terminal has no such limit.
For a real terminal, pair it in your Square dashboard and paste its Device ID. Switch to live by setting SQUARE_ENV=production + a live token.

Ticket Numbering
Tickets are generated automatically as a random 6-character code with 3 capital letters and 3 numbers (for example K7M2QX). The letter O and the digit 0 are never used, so codes can't be misread on a key tag or over the phone. Every code is checked to make sure it hasn't been handed out already.
Car Park Capacity
Number of open spots available to the valet (keep the reserved spots out of this count). Drives the dashboard Occupancy gauge. Synced to all devices.
Overstay Alert
A car on site longer than this many hours is flagged in the Active Cars overstay banner. (Cars left overnight — checked in before today — are always flagged.) Synced to all devices.
Car Wash — Xtream Costs
The customer pays the retail price (shown for reference). Enter your wholesale cost — what you owe Xtream per wash — so the Car Wash report can show your margin. Leave a cost blank if it isn't agreed yet. Synced to all devices; applied to new washes from now on.
Key-Tag Printer

Print the key tag (ticket #, bay, plate + scannable barcode) straight to a BIXOLON SPP-R200III. Because this is a web app, a small free bridge app must run on this device:

One-time setup on this device:
  1. Android: install “mPrintServer” from the Play Store. Windows: install the Web Print SDK utility from BIXOLON’s SPP-R200III download page.
  2. Pair the SPP-R200III to this device over Bluetooth, then add it in mPrintServer and give it a name. In its Web Server settings make sure the server is ON (port 18080).
  3. Type that exact name below, Save, then tap Test Print.
Leave Connection on HTTP and address as 127.0.0.1:18080 to start. If printing is blocked because this site is HTTPS, switch the WebPrint app to its secure mode and pick HTTPS here (adjust the port if it differs).

Print the key tag straight to a Zebra ZD421 connected by USB to this desktop. A small free Zebra app must be running on this PC:

One-time setup on this desktop:
  1. Install Zebra Browser Print (free) from zebra.com, then plug the ZD421 in over USB and turn it on.
  2. Open Browser Print — it should list the ZD421. Set it as the default printer.
  3. Because this site is HTTPS, visit https://localhost:9101 once in this browser and accept its security warning (this trusts Browser Print). Then tap Test Print below.
Labels are laid out for 2" × 3" stock at 203 dpi. If text sits off the label after a test print, tell me and I'll nudge the layout.
Customer SMS Credit

Customers are auto-texted their ticket link on check-in via ClickSend. If the ClickSend balance runs out, texts silently stop (the app falls back to opening your Messages app). Check it here and top up at dashboard.clicksend.com when it's low. You'll also get a warning banner at login whenever credit is low.

Data Management

Your data is stored locally in this browser. Back it up regularly by exporting to CSV or JSON. If you clear your browser data or use a different PC, you'll need to import the backup.

Automatic cloud backup

A full snapshot is saved to the cloud every night automatically and kept for 30 days — a safety net if data is changed or deleted by mistake. It stays server-side (never downloaded here unless you ask).
